解决GitHub Page 部署博客访问速度慢

  1. 2019-11-29
  2. 544字
  3. 2 min read

本站刚开始就是使用GitHub Page + Gridea 搭建的，还配置了自己的域名，但是由于Github是在国外服务器的原因，所以国内访问速度不是很理想。

所以我想了几个解决方法：

  1. 部署得到Coding
  2. Nginx 部署静态文件
  3. OSS 部署静态网站

先说第一个，我去腾讯开发者平台注册了个人账号，设置了配置静态页面，但是它的自动更新代码部署的功能太差了，手动更新还提示在更新，去社区找解决方案，回答是换企业版，那个好点。然后我换了企业版，注册登录，部署页面，没问题，但是连接Gridea客户端的时候出问题了，提示用户名或者仓库Token有问题，试了好几次，还是行，后来放弃了！

第二个Nginx 部署 和第三个 OSS 部署，这两个方式的最大麻烦之处在于需要手动更改页面中的路径，需要修改为相对路径，但是每次代码提交都需要改，还是放弃了。

在我一筹莫展的时候，看到一篇文章，说的是用Caddy代理GitHub Page，通过配置WebHook实现自动更新页面的需求 （需要有一台公网服务器）。

所以目前我的Blog解决方案就是： GitHub Page + Gridea + Caddy + Ngixn

  • Gridea 和 Github Page 解决写博客和存储博客的问题
  • Caddy 解决GitHub 的WebHook通知和直接更新部署静态页面的问题（支持自动申请免费证书）
  • NGINX 解决配置端口和绑定证书的问题 （需要去阿里或者腾讯申请免费证书部署）
  • OSS 负责图片和文件的存储，页面直接引入就好了
  • CDN 为域名和OSS开启加速

以上就是我的解决方案，整个弄下来涉及的东西也挺多的，但是好在配置并不复杂，而且访问速度还可以。

码云免费版不支持绑定域名，年费99 ，不知道Gridea能不能连接上。

如果你有好的解决方案欢迎留言😋！
