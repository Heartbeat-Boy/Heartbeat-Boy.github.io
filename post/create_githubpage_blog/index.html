使用GitHub Page + Gridea 搭建不用备案的博客

2018-12-12 / 3 min read

👏 欢迎进来观看这篇文章 ！
本站就是基于GitHub Page + Gridea 搭建的，域名买了备案还没完成，但是发现绑定自己的域名没有任何问题，就是访问的时候由于是GitHub所以访问速度就那样，后续会迁移到自己服务器上。

GitHub 上的操作

  1. 首先注册一个github账号，这个很简单，只要你拥有一个自己的邮箱，就可以轻松注册下来。
  2. 创建好以后，点击左侧 Repositories 旁边的 new 按钮
  3. 点击后接下来填写你的GitHub Page仓库的信息
  4. 填写完成后，你就可以去配置自己的GitHub page 地址了，首先点击 setting 按钮
  5. 然后就会进入仓库的设置页面，往下滑到GitHub Pages，然后在Source 这个下面点击 None 设置为 master branch ，这个是主分支的意思，因为博客没有复杂操作，所以都是上传到主分支。这边只要你选择了主分支，它机会刷新页面，刷新完拉下来就可以看到如下页面，你可以直接点击给你生成的页面链接。（如果点击链接他报错，可能是还没同不好，你稍微等2秒刷新就好了。）

    点击链接就会跳转到 http://heartbeat-boy.cn/test.github.io/ 因为我们在创建仓库的时候选择了初始化readme文件，所以这个页面上就会出现readme文件里的内容。
    设置自己的域名,在Custom domain 下填写自己的域名，这个时候点保存，会提示你解析错误。如下图：这是因为你没有在自己的域名解析添加解析记录，所以会报错。这个时候我们把Repository name写到解析记录如果你想使用www.域名的方式也可以再添加一条，只需要把@ 改为wwww就好了。这个时候刷新页面，就会提示你新的地址是你绑定的域名，推荐开启HTTPS，这样被备案监测的几率小。

Gridea客户端上的操作

  1. 首先去 Gridea 官网，点击下载适合自己客户端，目前支持 Mac Windows Ubuntu桌面版
  2. 接着打开软件，点击配置，然后填写信息，类似下图：域名如果自己已经绑定了就填自己的，如果没有就填写GitHub 提供的。CNAME 填写自己的域名，Token 需要去GitHub配置。
  3. 在GitHub页面，点击右上角自己的头像，然后点击 Developer settings
  4. 按照上图进行配置，Note 栏随便写，例如： blog。勾选 repo。然后下拉点击生成token。它会主动跳转到页面，然后复制显示的那个token就好了，
  5. 填好后点击检测连接，像我这样提示成功就好了，然后你可以点击左下角的本地预览，也可以点击同步，同步完成以后访问自己的域名看看有没有更新。
